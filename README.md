# CLI Expense Tracker

## Опис проєкту

CLI Expense Tracker — це консольна утиліта для управління особистими фінансами. Вона дозволяє:

* додавати витрати
* переглядати список витрат
* фільтрувати витрати за категорією та датами
* отримувати агреговану статистику

Дані зберігаються у форматі JSON (`expenses.json`) з наступною структурою:

```json
{
  "id": "uuid4-string",
  "date": "YYYY-MM-DD",
  "category": "string",
  "amount": float,
  "description": "string"
}
```

---

## ⚙️ Вимоги

* Python 3.8+
* Використовується тільки стандартна бібліотека

## 🚀 Інструкція запуску

### 1. Підготовка

Створіть файл:

```
expense_tracker.py
```

Або склонуйте репозиторій:

```bash
git clone <your-repo-url>
cd <project-folder>
```

---

### 2. Додавання витрати

```bash
python expense_tracker.py add --amount 100 --category Food --description "Lunch"
```

З датою:

```bash
python expense_tracker.py add --amount 200 --category Transport --date 2026-05-01
```

Параметри:

* `--amount` — сума (обов’язково, > 0)
* `--category` — категорія (обов’язково)
* `--description` — опис (опціонально)
* `--date` — дата у форматі `YYYY-MM-DD` (опціонально)

---

### 3. Перегляд списку витрат

```bash
python expense_tracker.py list
```

Фільтрація:

```bash
# За категорією
python expense_tracker.py list --category Food

# За датами
python expense_tracker.py list --start-date 2026-05-01 --end-date 2026-05-10

# Комбіновано
python expense_tracker.py list --category Food --start-date 2026-05-01
```

---

### 4. Перегляд звіту

```bash
python expense_tracker.py report
```

Приклад результату:

```
Звіт по витратах:
------------------------------
Food: 250.00
Transport: 120.00
------------------------------
Загалом: 370.00
```

## 📁 Зберігання даних

* Якщо файл відсутній — повертається порожній список і файл буде створений при першому записі
* Якщо файл пошкоджений — помилка обробляється без падіння програми

## 🧠 Архітектура

Проєкт складається з логічних шарів:

1. **CLI Layer (argparse)**
   Обробка команд:

   * `add`
   * `list`
   * `report`

2. **Data Layer**

   * `load_expenses()` — читання JSON
   * `save_expenses()` — запис JSON

3. **Business Logic**

   * `add_expense()` — створення запису (з UUID)
   * `filter_expenses()` — фільтрація за категорією і датами
   * `aggregate_by_category()` — агрегація
   * `display_expenses()` — табличне відображення
   * `generate_report()` — звіт

---

## 📊 Основні можливості

* Генерація унікального `id` для кожної витрати (UUID4)
* Фільтрація:

  * за категорією (точне співпадіння)
  * за діапазоном дат (включно)
* Табличний вивід витрат
* Агрегація сум за категоріями
